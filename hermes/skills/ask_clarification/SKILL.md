# ask_clarification

Задаёт тимлиду ровно ОДИН уточняющий вопрос, когда `task_parser` не может однозначно понять задачу.

## When to Use

- `task_parser` уверенно распознал факт постановки задачи, но не может выделить title (сообщение "сделай", "как договаривались", "тот баг" и т.п.).
- Между несколькими валидными интерпретациями нельзя выбрать без догадки.

**НЕ использовать**, чтобы:
- перепроверять дедлайн, если он явный;
- уточнять приоритет, если слово-маркер уже было;
- подтверждать очевидное.

## Hard cap

**Один круг уточнения на задачу. Точка.** После ответа пользователя собирай финальный JSON и передавай в `vikunja_create`, даже если всё ещё не идеально. Зацикливаться на диалоге — запрещено.

## Procedure

1. Сформулируй ОДИН короткий вопрос (одно предложение, без преамбул "извините, могу ли я…").
2. Через `send_message_tool` (из пресета `hermes-telegram`) отправь вопрос в тот же чат, из которого пришла задача.
   - `chat_id` уже в контексте текущего диалога.
   - `reply_to_message_id` — id исходного сообщения пользователя, если доступно.
3. Дождись ответа пользователя. После ответа:
   - слей ответ с исходным текстом в `description`;
   - перепройди priority/deadline-детекторы из `task_parser` по объединённому тексту;
   - вызови `vikunja_create`.
4. Если пользователь ответил неинформативно ("да", "не знаю", смайлик) — используй текущие данные, `priority: normal`, `deadline: null`, и всё равно создай задачу.

## Pitfalls

- **Никогда не задавай серию вопросов.** Один — и едем дальше.
- **Не отправляй уточнение, если запрос ясен.** Это раздражает тимлида. Лучше создать задачу и потом исправить, чем спамить вопросами.
- **Не спрашивай про приоритет и дедлайн**, если они не упоминались — проставляй дефолты (`normal`, `null`).
- **Не уходи в долгий свободный диалог.** Этот скилл — ровно для получения одного недостающего факта (обычно title).

## Verification

- В Telegram-логе только одно сообщение от бота (вопрос), затем создание задачи.
- В Vikunja появляется ровно одна задача, её title отражает уточнение пользователя.
